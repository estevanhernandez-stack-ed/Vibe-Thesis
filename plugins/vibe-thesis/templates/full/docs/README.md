# docs/

Project documentation for ThesisStudio itself — architecture, getting-started, contributing, troubleshooting, and Architecture Decision Records (ADRs).

## Files (populated as the build progresses)

| File | Purpose |
| --- | --- |
| `architecture.md` | The directory layout, mode flag, render pipeline, swarm orchestration |
| `getting-started.md` | Fork-to-first-chapter walkthrough (target: 15 min) |
| `contributing-guide.md` | How to file template-improvement issues, code style for scripts |
| `troubleshooting.md` | Common errors and resolutions |
| `adr/` | Architecture Decision Records — one per major technical choice |

## ADRs

Each ADR captures a single major decision with Status, Context, Decision, Consequences. Format follows [Michael Nygard's classic template](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions). See `adr/README.md` (forthcoming) for the pattern.
